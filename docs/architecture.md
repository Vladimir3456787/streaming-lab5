# Архитектура системы

## Почему выбрал Flink?
- Нужна низкая задержка P95 < 75 мс
- Flink обрабатывает данные сразу, а не ждет batch
- Подходит для реального времени

## Схема работы:
Данные → Kafka → Flink → ML модель → Результат

## Обработка опоздавших данных:
- Использую Watermark (отметки времени)
- Разрешаю данные опаздывать на 5 минут
- Сильно опоздавшие данные обрабатываю отдельно
